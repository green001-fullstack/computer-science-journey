# Why a tree is more than just a collection of values.

### A tree represents a relationship between values by storing the address of the children values

# What a node is.

### It is a small package that contains data and information about relationship 

# Why nodes need references to other nodes.

### To provide the relationship between itself and the other nodes

# Why Go uses *Node.

### *Node stores the memory address of another node.

# Why Java doesn't show *, even though the idea is similar.

### In java it is doing the same thing underneath but not explicitly defined
